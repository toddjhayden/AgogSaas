## Structure Standards

### File Organization
- Use descriptive filenames in kebab-case
- Group related documents in logical directories
- Include README.md in each directory
- Maintain consistent hierarchy
- Use relative links for navigation

### Document Structure
- Follow the [template](./template.md) structure
- Include version information
- Maintain table of contents
- Use consistent headings
- Include navigation links

### Content Guidelines
1. Clear and Concise
   - Use active voice
   - Write short paragraphs
   - Use bullet points for lists
   - Include relevant examples
   - Avoid jargon

2. Formatting
   - Use proper Markdown syntax
   - Consistent heading levels
   - Code block language specification
   - Table formatting
   - Image placement and sizing
   - Navigation Path (see [Navigation Path Standard](../../.github/NAVIGATION_PATH_STANDARD.md))
     * Top navigation shows full path: AGOG Home ‚Üí Parents ‚Üí Current Page
     * Bottom navigation with home and parent links
     * No old-style `[‚Üê Back]` links (replaced by bottom navigation)

3. Code Examples
   - Include language identifier
   - Use consistent formatting
   - Provide context
   - Include comments
   - Show input/output

## Maintenance

### Version Control

4. Link validation
5. Final approval

- Update triggers
- Deprecation process
- Archive procedure
